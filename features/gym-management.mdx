## Adding a Gym Member

Register students for gym membership programs with appropriate tier selection.

### Membership Duration Calculation

The system automatically calculates expiration dates based on membership type:

```php
// Bronze Member - 30 days
if($memberType == "bronzeMember") {
    $endingDate = date_create("now");
    date_add($endingDate, date_interval_create_from_date_string("30 days"));
    echo "Your membership will expire on " . date_format($endingDate, "Y-m-d");
}

// Silver Member - 60 days
elseif($memberType == "silverMember") {
    $endingDate = date_create("now");
    date_add($endingDate, date_interval_create_from_date_string("60 days"));
    echo "Your membership will expire on " . date_format($endingDate, "Y-m-d");
}

// Gold Member - 90 days
elseif($memberType == "goldMember") {
    $endingDate = date_create("now");
    date_add($endingDate, date_interval_create_from_date_string("90 days"));
    echo "Your membership will expire on " . date_format($endingDate, "Y-m-d");
}

// Diamond Member - 180 days
elseif($memberType == "diamondMember") {
    $endingDate = date_create("now");
    date_add($endingDate, date_interval_create_from_date_string("180 days"));
    echo "Your membership will expire on " . date_format($endingDate, "Y-m-d");
}
```

<Note>
  The expiration date is calculated and displayed immediately upon form submission.
</Note>

### Database Operation

```php
INSERT INTO Gym (Sid, gymFullName, userRegDate, memberType, medicalCondition)

```

## Viewing Gym Members

Access comprehensive gym membership information for all enrolled members.

### Display Columns

| Column | Description | Field Name |
|--------|-------------|------------|
| **Student ID** | Reference to student record | Sid |
| **Gym Member ID** | Unique gym membership identifier | gymMemberID |
| **Full Name** | Member's complete name | gymFullName |
| **Registration Date** | Date of enrollment | userRegDate |
| **Member Type** | Membership tier (Bronze/Silver/Gold/Diamond) | memberType |
| **Medical Condition** | Health considerations | medicalCondition |

### Database Query

```php
SELECT Sid, gymMemberID, gymFullName, userRegDate, memberType, medicalCondition 
FROM Gym
```

## Updating Gym Memberships

Modify existing gym membership records including tier changes and medical information updates.

### Update Process

<Steps>
  <Step title="Locate Member">
    Enter the Gym Member ID to find the record
  </Step>
  
  <Step title="Update Information">
    Modify:
    - Membership type (upgrade/downgrade)
    - Registration date (if renewing)
    - Medical condition information
    - Full name (if needed)
  </Step>
  
  <Step title="Recalculate Expiration">
    If changing membership type, the new expiration date will be calculated
  </Step>
  
  <Step title="Save Changes">
    Submit to update the gym membership record
  </Step>
</Steps>

### Database Operation

```php
UPDATE Gym 
SET gymFullName = '$gymFullName', userRegDate = '$userRegDate', 
    memberType = '$memberType', medicalCondition = '$medicalCondition' 
WHERE gymMemberID = '$gymMemberID'
```

## Deleting Gym Members

Remove gym membership records from the system.

### Deletion Process

1. Navigate to **Delete Gym Member** page
2. Enter the **Gym Member ID**
3. Confirm deletion

<Warning>
  This action permanently removes the gym membership record. Ensure the member is no longer actively using facilities.
</Warning>

### Database Operation

```php
DELETE FROM Gym 
WHERE gymMemberID = $gymMemberID
```

## Database Schema

The `Gym` table structure:

```sql
Table: Gym
- gymMemberID (Primary Key, Auto-increment)
- Sid (Foreign Key → Student1.Sid)
- gymFullName (VARCHAR)
- userRegDate (DATE)
- memberType (ENUM or VARCHAR: bronzeMember, silverMember, goldMember, diamondMember)
- medicalCondition (TEXT)
```
